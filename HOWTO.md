Project Description

This project is a simple Hospital Management System developed using Java and Object-Oriented Programming (OOP) concepts.
The purpose of the project is to simulate how a hospital system can manage information about patients, doctors, appointments, billing, and medical records.

The system allows users to enter patient information dynamically and displays the patient details. It demonstrates the use of classes, objects, constructors, and exception handling.

Project Structure

The project contains the following Java classes:

- Patient.java – Stores patient information such as name, age, and illness.
- Doctor.java – Represents doctor details.
- Appointment.java – Handles scheduling of appointments.
- Billing.java – Manages billing information for hospital services.
- MedicalRecords.java – Stores patient medical history and records.
- Main.java – The main class that runs the program and interacts with the user.

Features

- Object-Oriented Programming structure.
- Dynamic user input using "Scanner".
- Exception handling to prevent the program from crashing.
- Organized classes representing different hospital operations.

Exception Handling

Exception handling is implemented using try, catch, and finally blocks to handle possible errors during user input.

For example:

- If the user enters text instead of a number for age, the program catches the error.
- The program displays an error message instead of crashing.

This improves the reliability and effectiveness of the system

Technologies Used

- Java
- Object-Oriented Programming (OOP)
- GitHub for version control

How to Run the Program

1. Compile the Java files.
2. Run the "Main.java" file.
3. Enter the required patient information when prompted.

Example commands:

javac Main.java
java Main
